# TextDrop 开发日志

## 2026-05-17 初始化

- 项目目标：实现 TextDrop v0.1，Windows 端提供便携版 GUI，手机浏览器发送文本到电脑当前光标位置。
- 当前关键决策：Python + PySide6 + FastAPI/uvicorn + PyInstaller；服务监听 `0.0.0.0`；使用剪贴板 + Ctrl+V；Token 保存在 `%APPDATA%\TextDrop\config.json`。
- 风险记录：多网卡环境下自动选择地址可能不总是正确，因此 Windows 主界面需要提供地址下拉框作为兜底。
- 记录约束：开发日志只记录技术坑、实现判断、兼容性问题和后续想法，不记录用户发送正文。
- 实现记录：打包脚本依赖 `pyinstaller`，因此将其加入项目依赖，保证开发环境执行安装后即可直接打包。
- 验证记录：已完成语法编译、依赖导入、本地服务 `health/send` 冒烟测试、Token 错误测试、100KB 超限测试和 PyInstaller 打包测试。
- 打包结果：已生成 `dist\TextDrop.exe`，当前体积约 61 MB。
- 修正记录：手机端发送前连通性检查遇到 Token 失效时，需要提示“连接已失效，请重新扫码”，不能只显示普通发送失败。
- 兼容记录：手机语音输入可能受输入法组合态影响。已移除 `spellcheck=false`，为输入框补充移动端输入属性，并在发送前等待输入法提交当前内容。
- 兼容记录：手机端文本框需要尽量保持原生简单，已移除额外输入属性；底部发送栏改为固定在可视区域底部，并使用 `visualViewport` 在键盘弹起时上移。
- 后续想法：当前 PyInstaller 使用 `--onefile`，启动慢大概率来自单文件模式每次先解压 PySide6 等依赖。功能稳定后可考虑改为 `--onedir` 便携文件夹版本，以明显改善启动速度，代价是发布物从单个 exe 变成一个文件夹。
- 后续发布方案：优先考虑 `--onedir + zip`。即先打包成包含 `TextDrop.exe` 和依赖目录的文件夹，再压缩为 zip 发布；用户首次使用前解压整个文件夹后运行。该方案启动速度接近安装包，明显优于 `--onefile`，同时比安装包简单。README 需要提示不要单独移动 `TextDrop.exe`。
- 打包调整：已将打包脚本从 `--onefile` 改为 `--onedir + zip`，用于降低启动等待时间。输出目录为 `dist\TextDrop-v0.1-windows\TextDrop`，压缩包为 `dist\TextDrop-v0.1-windows.zip`。
- 修正记录：`onedir` 版本双击无窗口，是因为 PyInstaller 入口指向 `app.py`，但该文件只定义 `main()` 没有在脚本模式调用。已补充 `if __name__ == "__main__": main()`。
- 修正记录：`--windowed` 打包后 uvicorn 默认日志配置会访问空的 `sys.stderr`，导致 `Unable to configure formatter 'default'`。已在 uvicorn 配置中设置 `log_config=None`。
- 界面调整：手机端连接状态从底部发送按钮左侧移动到顶部左侧；顶部布局为左侧连接状态、中间产品名、右侧清空按钮；底部改为整行大发送按钮。
- 界面调整：英文连接失败状态 `Disconnected` 在顶部空间内容易被截断，改为更短的 `Offline`，并适当加宽顶部左侧状态区域。
- 界面调整：手机端顶部产品名改为绝对居中，避免受左侧连接状态和右侧清空按钮宽度影响。
- 需求同步：发布形态确定为 `onedir + zip`，不是单文件 exe。粘贴后自动按 Enter / 自动提交目标应用已实现为可选功能（2026-05-17），默认关闭。
- 图标处理：根目录图标源图为真实透明 PNG，原始大小约 1.14 MB。已生成 `assets\app_icon.png`（约 531 KB）和多尺寸 `assets\app_icon.ico`（约 65 KB），并接入窗口图标与 PyInstaller 打包图标。
- 图标修正：标题栏图标显得过小，原因是源图透明边距较大且小尺寸会压缩细节。已裁切透明边距并重新生成图标，`assets\app_icon.png` 约 661 KB，`assets\app_icon.ico` 约 86 KB；同时补充 QApplication 图标和 Windows AppUserModelID，减少任务栏使用旧图标的概率。
- 宣传文案：已完成三版宣传文案初稿——公众号微信推文、README 重构、社交媒体短文案。核心卖点聚焦「不绑定任何输入法」和「安静环境下小声语音输入→电脑」两个痛点。文案已存入思源笔记 `/AI实践/TextDrop介绍/`。
- 文案定位：产品核心痛点确认为：办公室/图书馆等安静场所无法大声语音输入，电脑麦克风距离远效果差；现有跨设备方案绑定特定输入法。TextDrop 解决的是「任意输入法 + 局域网秒传 + 小声说话」的细分场景。
- 后续想法：README 需要从纯技术说明转型为「产品亮点 + 快速开始」结构，配合 Release 页面使用。当前 README 仍偏技术向，待整合。
- 新功能（2026-05-17）：粘贴后自动按键。Windows 主界面"语言"下方新增"粘贴后按键"下拉框，选项包括不自动按键、Enter、Ctrl+Enter、Shift+Enter、Alt+Enter，默认不自动按键。配置持久化到 config.json 的 `auto_enter` 字段。实现：paste_text 新增 auto_enter 参数，粘贴完成后按需调用 pyautogui.hotkey 模拟组合键；ServerState 新增 get_auto_enter 回调，每次发送时读取当前配置。
- 后续规划（2026-05-17）：macOS 适配。目标是一套代码双平台出包，Release 同时发布 Windows zip 和 macOS dmg/app。涉及配置路径、快捷键映射、网卡检测、打包脚本四个模块的平台差异处理。初步分析：配置用 platformdirs 统一；快捷键用 platform.system() 判断；网卡关键词需要增加 macOS 特有虚拟接口过滤；打包不可交叉编译，需要 GitHub Actions matrix build 或手动双平台分别构建。
- 新功能（2026-05-27）：手机端底部按钮组改为左右“投递并发送”、中间“投递”。“投递”只粘贴文本，不应用 Windows 端追加按键；“投递并发送”粘贴后按当前追加按键设置执行。API `POST /api/send` 新增 `apply_auto_enter` 布尔字段，旧请求默认继续应用追加按键。
- 连接优化（2026-05-27）：Windows 端新增 3 秒定时检测本机地址变化。当前二维码地址不再可用时自动重新选择地址、刷新访问地址和二维码，并在主界面提示用户重新扫码。后台 uvicorn 线程不存活时会尝试重启服务并刷新界面提示。
- 连接状态（2026-05-27）：Windows 主界面状态从“正在运行/已关闭”调整为手机连接状态“已连接/未连接”。有效 Token 的 `/api/health` 或 `/api/send` 请求会刷新最近连接时间；超过短时间未收到手机请求、地址切换或 Token 刷新后显示未连接。
- 版本调整（2026-05-27）：当前测试发布版本号调整为 `0.1.2`，发布包名称调整为 `TextDrop-v0.1.2-windows.zip`。
